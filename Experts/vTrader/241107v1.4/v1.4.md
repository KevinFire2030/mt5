int CheckEmaSignal() {
    double ema5[], ema20[], ema40[];
    ArraySetAsSeries(ema5, true);
    ArraySetAsSeries(ema20, true);
    ArraySetAsSeries(ema40, true);
    
    CopyBuffer(m_hEma5, 0, 0, 1, ema5);
    CopyBuffer(m_hEma20, 0, 0, 1, ema20);
    CopyBuffer(m_hEma40, 0, 0, 1, ema40);
    
    Print("=== EMA 데이터 ===");
    Print("EMA5: ", ema5[0]);
    Print("EMA20: ", ema20[0]);
    Print("EMA40: ", ema40[0]);
    
    double minDiff = 0.5;  // 최소 차이 설정
    
    // 정배열 체크 (5 > 20 > 40)
    if(ema5[0] > ema20[0] + minDiff && ema20[0] > ema40[0] + minDiff) {
        Print("=== EMA 정배열 ===");
        Print("EMA5-20 차이: ", ema5[0] - ema20[0]);
        Print("EMA20-40 차이: ", ema20[0] - ema40[0]);
        return 1;
    }
    
    // 역배열 체크 (5 < 20 < 40)
    if(ema5[0] + minDiff < ema20[0] && ema20[0] + minDiff < ema40[0]) {
        Print("=== EMA 역배열 ===");
        Print("EMA20-5 차이: ", ema20[0] - ema5[0]);
        Print("EMA40-20 차이: ", ema40[0] - ema20[0]);
        return -1;
    }
    
    Print("=== EMA 중립 ===");
    return 0;
}


현재 코드 동작
현재 OnTick 함수는 다음과 같은 조건을 가지고 있습니다:
!m_isAutoTrading: 자동 매매가 활성화되지 않은 경우, 함수가 즉시 종료됩니다.
!IsNewBar(): 새로운 봉이 생성되지 않은 경우, 함수가 즉시 종료됩니다. 즉, 새로운 봉이 생성될 때만 거래 신호를 확인하고 포지션을 오픈하거나 청산합니다.
!IsNewBar() 주석 처리 시 동작
!IsNewBar() 조건을 주석 처리하면, OnTick 함수는 매 틱마다 실행됩니다. 즉, 새로운 봉이 생성될 때뿐만 아니라, 모든 틱에서 거래 신호를 확인하고 포지션을 오픈하거나 청산할 수 있습니다.
결과
장점: 더 많은 거래 기회를 포착할 수 있습니다. 시장의 미세한 움직임에 반응하여 더 자주 거래할 수 있습니다.
단점: 너무 많은 거래가 발생할 수 있으며, 거래 비용이 증가할 수 있습니다. 또한, 시장의 노이즈에 의해 잘못된 신호가 발생할 가능성이 높아집니다.
따라서, !IsNewBar() 조건을 주석 처리할 경우, EA의 거래 전략이 실시간 틱 데이터에 적합한지 신중히 고려해야 합니다. 실시간 틱 단위로 거래하는 것은 더 많은 리소스를 소모할 수 있으며, 전략의 성능에 영향을 미칠 수 있습니다.
